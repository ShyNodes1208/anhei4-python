# 暗黑4 魔盒自动化 — 使用说明

> Python + 图像识别版  
> 适用分辨率：**2560 × 1440**（全屏或无边框窗口）

---

## 一、功能简介

本工具用于自动化完成暗黑破坏神4 **赫拉迪姆魔盒** 的三步批量操作：

| 阶段 | 操作 | 说明 |
|------|------|------|
| 1 | 添加词缀 | 将装备补满到 4 条词缀 |
| 2 | 升级至传奇 | 将装备升级为传奇品质 |
| 3 | 嬗变物品 | 对装备执行嬗变 |

**执行方式（方案 B：逐阶段）**

```
全部装备 添加词缀 → 全部装备 升级至传奇 → 全部装备 嬗变
```

**智能跳过**

- 装备不符合配方要求时，屏幕会出现红字 **「该物品不适用于此配方组合」**
- 脚本识别到该提示后**自动跳过该件**，不会误点「重置暗金威能」等按钮
- 跳过后点击 **「清除」** 按钮清空魔盒，不影响后续装备

**动态配方定位**

- 放入不同装备后，配方列表位置会重排
- 脚本通过图像识别动态找到「添加词缀 / 升级至传奇 / 嬗变物品」按钮
- 识别失败时退回固定坐标兜底

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / 11 |
| Python | 3.10 及以上 |
| 游戏分辨率 | **2560 × 1440** |
| 游戏窗口 | 全屏 或 无边框窗口（推荐） |

### 1. 克隆或下载项目

```bash
git clone https://github.com/ShyNodes1208/anhei4-python.git
```


## 四、首次使用（必做）

### 4.1 确认模板图

`templates/` 目录下应包含以下识别模板：

| 文件名 | 用途 |
|--------|------|
| `accept.png` | 「接受」按钮 |
| `not_applicable.png` | 红字「该物品不适用于此配方组合」 |
| `recipe_add_affix.png` | 配方「添加词缀」 |
| `recipe_upgrade.png` | 配方「升级至传奇」 |
| `recipe_transmute.png` | 配方「嬗变物品」 |

若模板缺失或识别不准，可用 `d4_capture.py` 重新截取（见 [第八章](#八辅助工具)）。

### 4.2 校准坐标

打开游戏，进入魔盒界面，然后运行：

```bash
python d4_auto.py --calibrate
```

脚本会在 3 秒后依次将鼠标移动到各关键位置。请观察鼠标是否准确停在：

- 背包第 1 格 / 最后一格
- 添加词缀、升级至传奇、嬗变物品（固定坐标参考点）
- 重塑、接受、**清除**、安全位置

若某点偏移，在 `d4_config.py` 中修改对应坐标。

### 4.3 小范围试跑（强烈建议）

首次正式运行前，先用 **1 格** 测试：

1. 打开 `d4_config.py`，设置：
   ```python
   ROWS = 1
   COLS = 1
   ```
2. 背包第一格放入 **1 件** 待处理装备
3. 运行脚本，按 F8 启动，观察三阶段是否全部正确

确认无误后，再将 `ROWS`、`COLS` 改回实际背包范围。

## 五、日常使用

### 5.1 启动脚本

```bash
python d4_auto.py
```

控制台会显示当前配置，例如：

```
==================================================
 暗黑4 魔盒一键全流程（Python 图像识别版）
 处理范围：3 行 x 8 列
 阶段：加词缀=True 升传奇=True 嬗变=True
 开始键: F8    停止键: F10
==================================================
切到游戏，打开魔盒+背包，按 F8 开始……
```

### 5.2 游戏内准备

1. 打开 **赫拉迪姆魔盒** 界面
2. 打开 **背包**（装备页）
3. 将待处理装备放入背包，从 **第 1 格开始**，**从左到右、从上到下** 排列
4. 确保魔盒处于空盒状态（无残留装备）

### 5.3 开始 / 停止

| 按键 | 作用 |
|------|------|
| **F8** | 开始执行（按一下） |
| **F10** | 停止（运行中按下） |
| **ESC** | 在控制台窗口按 ESC 退出程序 |

> 按 F8 后有约 1 秒启动延迟，请在此时间内切回游戏窗口。

### 5.4 运行日志

终端会实时输出处理进度，例如：

```
[12:50:56] ==== 阶段开始：添加词缀，共16格 ====
[12:50:56] 【加词缀】第1格 (1722,1017)
[12:50:58]   找到 recipe_add_affix.png @(1132,328) 置信0.98
[12:51:05] 【加词缀】第2格 (1795,1017)
[12:51:07]   -> 不适用，跳过该件
...
[12:55:00] ==== 阶段完成：添加词缀（成功14，跳过2）====
```

- **成功**：该件正常完成当前阶段
- **跳过**：该件不适用当前配方，已自动跳过
- **停止**：用户按 F10 中断

## 六、配置说明

所有可调参数在 **`d4_config.py`** 中。

### 6.1 背包处理范围

```python
ROWS = 3   # 处理几行
COLS = 8   # 每行几列
```

脚本按 `ROWS × COLS` 个格子，从左到右、从上到下依次处理。  
**每次运行前按实际放入的装备数量修改。**

### 6.2 阶段开关

```python
DO_ADD_AFFIX = True   # 阶段1：添加词缀
DO_UPGRADE   = True   # 阶段2：升级至传奇
DO_TRANSMUTE = True   # 阶段3：嬗变物品
```

调试时可单独关闭某一阶段，例如只测嬗变：

```python
DO_ADD_AFFIX = False
DO_UPGRADE   = False
DO_TRANSMUTE = True
```

### 6.3 添加词缀次数

```python
AFFIX_REFORGE_COUNT = 4
```

表示连点「重塑」的总次数。第一次会弹出绑定确认并点「接受」，之后继续重塑补满。  
加满 4 条后多余的点击无效，不影响后续。

### 6.4 时间参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `START_DELAY` | 1.0 秒 | 按 F8 后等待再开始 |
| `STEP_DELAY` | 0.8 秒 | 每步点击后等待 |
| `BETWEEN_ITEMS_DELAY` | 0.8 秒 | 每件装备之间等待 |
| `BETWEEN_PHASES_DELAY` | 0.8 秒 | 阶段切换之间等待 |
| `ACCEPT_WAIT_TIMEOUT` | 1.8 秒 | 等待「接受」按钮的最长时间 |
| `DETECT_AFTER_PLACE` | 0.6 秒 | 放入装备后等待再检测 |

若点击过快导致界面未刷新，可适当**增大** `STEP_DELAY`。  
若运行太慢，可适当**减小**（不建议低于 0.5 秒）。

### 6.5 热键

```python
START_KEY = "f8"
STOP_KEY  = "f10"
```

可改为其他键名（如 `"f9"`），须为 `keyboard` 库支持的键名。

### 6.6 识别阈值

```python
MATCH_THRESHOLD = 0.80      # 接受、不适用等模板
RECIPE_THRESHOLD = 0.72     # 配方按钮（跨选中/未选中状态，略放宽）
```

识别不稳定时可适当**降低**（如 0.72 / 0.65）；误识别时可**提高**。

## 七、工作流程详解

### 阶段 1：添加词缀（每件装备）

```
1. 动态找到「添加词缀」配方并点击
2. 右键背包装备 → 放入魔盒
3. 检测红字「不适用」→ 是则清除并跳过
4. 连点「重塑」AFFIX_REFORGE_COUNT 次（首次会点「接受」）
5. 点击「清除」取回装备
```

### 阶段 2：升级至传奇（每件装备）

```
1. 右键背包装备 → 放入魔盒
2. 动态找到「升级至传奇」配方并点击
3. 检测红字「不适用」→ 是则清除并跳过
4. 点击「重塑」→ 等待并点击「接受」
5. 点击「清除」取回装备
```

### 阶段 3：嬗变物品（每件装备）

```
1. 右键背包装备 → 放入魔盒
2. 动态找到「嬗变物品」配方并点击
3. 检测红字「不适用」→ 是则清除并跳过
4. 点击「重塑」→ 等待并点击「接受」
5. 点击「清除」取回装备
```

```bash

```

鼠标依次移动到各预设坐标，用于肉眼检查是否准确。

```bash
# 1. 在游戏里摆好目标画面，回到终端按回车截全屏
```

模板要求：

- 在 **2560×1440** 下截取
- 只裁稳定不变的小块（文字、按钮），不要裁会变化的数字或图标

```bash

```

立即保存一张全屏截图 `shot_YYYYMMDD_HHMMSS.png`，便于排查问题。

## 九、常见问题

### Q1：按 F8 没反应

- 终端是否**以管理员身份**运行？
- 脚本是否在运行中（控制台有无「切到游戏…」提示）？
- 是否误在游戏窗口按了 F8（应在脚本运行后任意时刻按，但需 `keyboard` 能捕获）？

### Q2：鼠标点击位置偏移

- 确认游戏为 **2560×1440**，且 Windows 显示缩放为 **100%**
- 运行 `python d4_auto.py --calibrate` 检查各点
- 在 `d4_config.py` 中微调对应坐标

### Q3：识别不到「接受」或红字，导致流程卡住

- 检查 `templates/` 下模板是否存在且清晰
- 用 `python d4_locate.py accept.png` 测试能否找到
- 适当降低 `MATCH_THRESHOLD`（如 0.75）
- 检查 `REGION_ACCEPT`、`REGION_NOT_APPLICABLE` 是否覆盖目标区域

### Q4：配方点错了

- 用 `python d4_locate.py recipe_upgrade.png` 等测试
- 若置信度低于 `RECIPE_THRESHOLD`，会退回固定坐标；可检查 `UPGRADE_XY` 等兜底坐标
- 配方列表变化较大时，可能需从 `start.png` 重新裁 `recipe_*.png` 模板

### Q5：4 词条传奇被误处理 / 跳过逻辑不对

- 4 词条传奇无法添加词缀，脚本应识别红字并 **跳过**，不会误点「重置暗金威能」
- 若仍误操作，检查 `not_applicable.png` 模板和 `REGION_NOT_APPLICABLE` 区域

### Q6：运行中报 `mss` 或线程相关错误

- 确保使用最新版脚本（F8 已在独立线程运行，避免 `mss` 跨线程问题）
- 不要同时运行多个实例

### Q7：想只跑某一个阶段

在 `d4_config.py` 中关闭不需要的阶段，例如只嬗变：

```python
DO_ADD_AFFIX = False
DO_UPGRADE   = False
DO_TRANSMUTE = True
```

## 十一、注意事项

1. **分辨率**：脚本按 2560×1440 编写，其他分辨率需重新校准。
2. **背包排列**：装备必须从第 1 格起连续排列，顺序为从左到右、从上到下。
3. **材料**：确保魔盒材料充足，脚本不会检测材料数量。
4. **停止**：随时按 **F10** 可中断；中断后请手动检查魔盒是否有残留装备。
5. **合规风险**：自动化可能违反游戏服务条款，请自行承担使用风险。
